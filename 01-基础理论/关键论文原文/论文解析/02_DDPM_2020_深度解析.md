## 研究背景

### 1.1 生成模型的演进

| 时期 | 代表模型 | 特点 | 问题 |
|------|---------|------|------|
| **2014-2016** | GAN | 对抗训练 | 训练不稳定，mode collapse |
| **2016-2018** | VAE | 变分推断 | 生成模糊 |
| **2018-2020** | Flow | 精确似然 | 计算复杂度高 |
| **2020-** | Diffusion | 去噪生成 | 采样慢但稳定 |

### 1.2 GAN 的困境

```
GAN 训练问题:
┌─────────────┐     ┌─────────────┐
│  生成器 G   │ ←→  │  判别器 D   │
└─────────────┘     └─────────────┘
     ↑                    ↑
  生成假图              判别真假

问题:
1. 训练不稳定：G 和 D 需要平衡
2. Mode Collapse: G 只生成几种样本
3. 难以训练：需要精细调参
```

### 1.3 物理启发

DDPM 受到**非平衡热力学**的启发：

```
热力学过程:
1. 加热：有序 → 无序 (加噪)
2. 冷却：无序 → 有序 (去噪)

类比到图像生成:
1. 前向过程：清晰图像 → 纯噪声
2. 反向过程：纯噪声 → 清晰图像
```

## 核心问题

### 2.1 核心挑战

**如何生成高质量图像，同时避免 GAN 的训练不稳定问题？**

### 2.2 关键洞察

```
传统思路：直接从噪声生成图像 (困难)
DDPM 思路：逐步去噪，每次只做一点 (简单)

类比:
- 直接生成：像凭空画一幅画
- 逐步去噪：像从模糊到清晰慢慢雕琢
```

### 2.3 核心思想

```
1. 定义前向扩散过程 (固定)
   x₀ → x₁ → x₂ → ... → x_T (逐步加噪)

2. 学习反向去噪过程 (可学习)
   x_T → x_{T-1} → ... → x₀ (逐步去噪)

3. 关键：每一步都很简单，组合起来很强大
```

## 核心贡献

### 3.1 扩散概率模型框架

```
前向过程 (扩散):
q(x_t | x_{t-1}) = N(x_t; √(1-β_t)x_{t-1}, β_t I)

反向过程 (去噪):
p_θ(x_{t-1} | x_t) = N(x_{t-1}; μ_θ(x_t, t), Σ_θ(x_t, t))

其中:
- β_t: 噪声调度 (固定)
- μ_θ, Σ_θ: 神经网络学习
```

### 3.2 预测噪声而非原图

**这是 DDPM 最关键的洞察！**

```
传统思路：预测原图 x₀
DDPM 思路：预测噪声 ε

损失函数:
L = E[||ε - ε_θ(x_t, t)||²]

为什么有效？
1. 噪声是简单的高斯分布
2. 原图是复杂的数据分布
3. 学习简单分布更容易
```

### 3.3 任意时刻直接采样

```
关键公式:
q(x_t | x_0) = N(x_t; √ᾱ_t x_0, (1-ᾱ_t)I)

其中 ᾱ_t = ∏_{s=1}^t (1-β_s)

好处:
- 可以直接从 x_0 计算任意 t 的 x_t
- 不需要一步步加噪
- 训练时随机采样 t，效率高
```

## 数学原理

### 4.1 前向扩散过程

```
定义:
q(x_t | x_{t-1}) = N(x_t; √(1-β_t)x_{t-1}, β_t I)

递推关系:
x_t = √(1-β_t) x_{t-1} + √β_t ε_{t-1},  ε_{t-1} ~ N(0, I)

展开:
x_t = √(1-β_t) [√(1-β_{t-1}) x_{t-2} + √β_{t-1} ε_{t-2}] + √β_t ε_{t-1}
    = √[(1-β_t)(1-β_{t-1})] x_{t-2} + ...

定义 α_t = 1 - β_t, ᾱ_t = ∏_{s=1}^t α_s

得到:
x_t = √ᾱ_t x_0 + √(1-ᾱ_t) ε,  ε ~ N(0, I)
```

### 4.2 反向去噪过程

```
目标：学习 p_θ(x_{t-1} | x_t)

假设:
p_θ(x_{t-1} | x_t) = N(x_{t-1}; μ_θ(x_t, t), σ_t² I)

其中 σ_t² 可以固定，只学习 μ_θ

根据贝叶斯定理:
q(x_{t-1} | x_t, x_0) = N(x_{t-1}; μ̃_t, σ̃_t² I)

其中:
μ̃_t = (√α_t (1-ᾱ_{t-1}) / (1-ᾱ_t)) x_t + (√ᾱ_{t-1} β_t / (1-ᾱ_t)) x_0
σ̃_t² = ((1-ᾱ_{t-1}) / (1-ᾱ_t)) β_t
```

### 4.3 损失函数推导

```
原始 ELBO:
L = E_q[log p(x_0) - log q(x_1:T | x_0)]
  = E_q[log p(x_T) + Σ log p(x_{t-1}|x_t) - Σ log q(x_t|x_{t-1})]

简化后:
L = E_t,x_0,ε[||ε - ε_θ(x_t, t)||²]

其中:
- t ~ Uniform(1, T)
- x_0 ~ 数据集
- ε ~ N(0, I)
- x_t = √ᾱ_t x_0 + √(1-ᾱ_t) ε
```

### 4.4 噪声调度

```
线性调度 (DDPM 使用):
β_t = β_1 + (t-1)/(T-1) × (β_T - β_1)
β_1 = 10⁻⁴, β_T = 0.02

余弦调度 (改进):
β_t = 1 - (f(t) / f(0))²
f(t) = cos((t/T + s) / (1+s) × π/2)
s = 0.008

效果:
- 线性：简单，效果好
- 余弦：早期加噪更慢，质量略好
```

## 模型架构

### 5.1 UNet 架构

```
DDPM 使用改进的 UNet:

输入：x_t (带噪图像) + t (时间步)
输出：ε_θ (预测的噪声)

架构:
x_t → Conv → Time Embedding → DownSample × 4
                              → Middle
                              → UpSample × 4
    → Conv → ε_θ

特点:
- 残差连接
- 注意力机制 (在 16×16 分辨率)
- GroupNorm
- SiLU 激活
```

### 5.2 时间嵌入

```
时间步 t 的编码:

1. Sinusoidal 位置编码 (类似 Transformer):
   e_t = [sin(ω_1 t), cos(ω_1 t), ..., sin(ω_d t), cos(ω_d t)]
   其中 ω_i = 10000^(-2i/d)

2. MLP 投影:
   emb = MLP(e_t)

3. 添加到 UNet 各层:
   feature = feature + emb
```

### 5.3 注意力机制

```
在 16×16 分辨率使用自注意力:

Attention(Q, K, V) = softmax(QK^T / √d) V

其中:
Q = W_Q · x, K = W_K · x, V = W_V · x

作用:
- 捕捉长程依赖
- 改善全局结构
```

## 训练方法

### 6.1 训练算法

```
Algorithm 1: DDPM 训练

Input: 数据集 {x₀⁽¹⁾, ..., x₀⁽ᴺ⁾}, T, 学习率 η
Initialize: UNet 参数 θ

repeat
    # 采样一个 batch
    {x₀⁽¹⁾, ..., x₀⁽ᴹ⁾} ~ 数据集
    
    for each x₀⁽ⁱ⁾ in batch:
        # 随机采样时间步
        t ~ Uniform({1, ..., T})
        
        # 采样噪声
        ε ~ N(0, I)
        
        # 计算带噪图像
        x_t = √ᾱ_t x₀⁽ⁱ⁾ + √(1-ᾱ_t) ε
        
        # 预测噪声
        ε_θ = UNet(x_t, t)
        
        # 计算损失
        L = ||ε - ε_θ||²
    
    # 平均损失
    L = 1/M Σ L⁽ⁱ⁾
    
    # 反向传播
    ∇_θ L = Backprop(L)
    
    # 更新参数
    θ ← θ - η ∇_θ L
    
until 收敛
```

### 6.2 采样算法

```
Algorithm 2: DDPM 采样

Input: 训练好的 UNet 参数 θ, T
Output: 生成的图像 x₀

# 从纯噪声开始
x_T ~ N(0, I)

for t = T, T-1, ..., 1:
    # 采样噪声
    z ~ N(0, I)  (如果 t > 1, 否则 z = 0)
    
    # 计算均值
    μ_θ = 1/√α_t (x_t - β_t/√(1-ᾱ_t) ε_θ(x_t, t))
    
    # 计算方差
    σ_t² = β_t
    
    # 采样
    x_{t-1} = μ_θ + σ_t z

return x₀
```

### 6.3 训练技巧

| 技巧 | 说明 | 效果 |
|------|------|------|
| **梯度裁剪** | 限制梯度范数 | 稳定训练 |
| **EMA** | 指数移动平均 | 改善生成质量 |
| **混合精度** | 使用 FP16 | 加速训练 |
| **学习率 warmup** | 逐步增加学习率 | 稳定初期训练 |

## 代码实现

### 8.1 噪声调度

```python
class NoiseScheduler:
    def __init__(self, T=1000, beta_start=1e-4, beta_end=0.02):
        self.T = T
        self.betas = torch.linspace(beta_start, beta_end, T)
        self.alphas = 1. - self.betas
        self.alphas_cumprod = torch.cumprod(self.alphas, dim=0)
    
    def add_noise(self, x0, noise, t):
        """一次性添加 t 步的噪声"""
        sqrt_alpha_cumprod = torch.sqrt(self.alphas_cumprod[t])
        sqrt_one_minus_alpha = torch.sqrt(1 - self.alphas_cumprod[t])
        return sqrt_alpha_cumprod * x0 + sqrt_one_minus_alpha * noise
    
    def get_loss_weight(self, t):
        """可选：加权损失"""
        return 1. / (1 - self.alphas_cumprod[t])
```

### 8.3 训练循环

```python
def train_ddpm(model, scheduler, dataloader, optimizer, device, epochs=10):
    model.train()
    
    for epoch in range(epochs):
        total_loss = 0
        
        for batch_idx, (x0, _) in enumerate(dataloader):
            x0 = x0.to(device)
            optimizer.zero_grad()
            
            # 随机采样时间步
            t = torch.randint(0, scheduler.T, (x0.shape[0],)).to(device)
            
            # 采样噪声
            noise = torch.randn_like(x0)
            
            # 加噪
            xt = scheduler.add_noise(x0, noise, t)
            
            # 预测噪声
            noise_pred = model(xt, t)
            
            # 计算损失
            loss = F.mse_loss(noise_pred, noise)
            
            # 反向传播
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
            optimizer.step()
            
            total_loss += loss.item()
        
        avg_loss = total_loss / len(dataloader)
        print(f'Epoch {epoch+1}: Loss = {avg_loss:.4f}')
```

### 8.4 采样函数

```python
@torch.no_grad()
def sample_ddpm(model, scheduler, shape=(1, 3, 32, 32), device='cuda'):
    model.eval()
    
    # 从纯噪声开始
    x = torch.randn(shape).to(device)
    
    for t in reversed(range(scheduler.T)):
        t_tensor = torch.tensor([t]).expand(shape[0]).to(device)
        
        # 预测噪声
        noise_pred = model(x, t_tensor)
        
        # 计算均值
        alpha = scheduler.alphas[t]
        alpha_cumprod = scheduler.alphas_cumprod[t]
        beta = scheduler.betas[t]
        
        x = (x - beta / torch.sqrt(1 - alpha_cumprod) * noise_pred) / torch.sqrt(alpha)
        
        # 添加噪声 (除了最后一步)
        if t > 0:
            z = torch.randn_like(x)
            x += torch.sqrt(beta) * z
    
    return x
```

## 深入理解

### 9.1 为什么预测噪声有效？

```
直觉解释 1: 简单 vs 复杂
- 噪声 ε: 简单的高斯分布 N(0, I)
- 原图 x₀: 复杂的数据分布
- 学习简单分布更容易

直觉解释 2: 残差学习
- x_t = √ᾱ_t x₀ + √(1-ᾱ_t) ε
- 预测 ε 相当于预测 x_t 和 x₀ 的差异
- 残差学习通常更容易

直觉解释 3: 信噪比
- 早期 t 小：x_t 接近 x₀，噪声小
- 后期 t 大：x_t 接近纯噪声，噪声大
- 预测噪声在整个过程中都合理
```

### 9.2 扩散过程的物理类比

```
热力学类比:
加热 (扩散): 冰块 → 水 → 水蒸气 (有序 → 无序)
冷却 (去噪): 水蒸气 → 水 → 冰块 (无序 → 有序)

图像生成:
扩散：清晰图像 → 模糊图像 → 纯噪声
去噪：纯噪声 → 模糊图像 → 清晰图像

关键:
- 加热过程是固定的 (物理定律)
- 冷却过程需要学习 (如何结晶)
```

### 9.3 与去噪自编码器的关系

```
去噪自编码器 (DAE):
输入：带噪图像 x_noisy
输出：去噪图像 x_clean
损失：||x_clean - DAE(x_noisy)||²

DDPM:
输入：带噪图像 x_t + 时间步 t
输出：预测噪声 ε_θ
损失：||ε - ε_θ(x_t, t)||²

关系:
- DDPM 可以看作 T 个 DAE 串联
- 每个 DAE 负责一个噪声水平
- 时间步 t 告诉 DAE 当前噪声水平
```

### 9.4 隐空间的理解

```
VAE 的隐空间:
x₀ → 编码器 → z (一次编码)
z → 解码器 → x₀' (一次解码)

DDPM 的"隐空间":
x₀ → x₁ → x₂ → ... → x_T (T 步编码)
x_T → x_{T-1} → ... → x₀ (T 步解码)

区别:
- VAE: 单层隐变量
- DDPM: 多层隐变量 (每个 x_t 都是一个层次)
- DDPM 更精细，但计算量更大
```

## 为什么预测噪声

### 10.1 理论分析

```
预测 x₀ 的损失:
L_x0 = E[||x₀ - x̂₀||²]

预测 ε 的损失:
L_ε = E[||ε - ε̂||²]

关系:
x̂₀ = (x_t - √(1-ᾱ_t) ε̂) / √ᾱ_t

当 t 接近 T 时:
- √ᾱ_t → 0
- 预测 x₀ 会放大误差
- 预测 ε 更稳定
```

### 10.2 实验对比

| 预测目标 | FID | 训练稳定性 |
|---------|-----|-----------|
| 噪声 ε | 9.4 | 稳定 |
| 原图 x₀ | 12.1 | 较稳定 |
| 速度 v | 9.8 | 稳定 |

**结论**: 预测噪声效果最好

### 10.3 梯度分析

```
预测 x₀:
∂L/∂θ = ∂||x₀ - x̂₀||²/∂θ
      = 2(x₀ - x̂₀) · ∂x̂₀/∂θ

预测 ε:
∂L/∂θ = ∂||ε - ε̂||²/∂θ
      = 2(ε - ε̂) · ∂ε̂/∂θ

当 t 大时:
- x₀ - x̂₀ 可能很大 (因为 x_t 接近纯噪声)
- ε - ε̂ 始终在合理范围
- 预测 ε 梯度更稳定
```

## 与 VAE 的关系

### 11.1 理论联系

```
VAE 的 ELBO:
log p(x) ≥ E_q[log p(x|z)] - KL(q(z|x) || p(z))

DDPM 的 ELBO:
log p(x₀) ≥ E_q[log p(x₀|x₁)] - KL(q(x₁|x₀) || p(x₁))
           + E_q[log p(x₁|x₂)] - KL(q(x₂|x₁) || p(x₂))
           + ...

DDPM 可以看作 T 层 VAE!
```

### 11.2 架构对比

| 特性 | VAE | DDPM |
|------|-----|------|
| **隐变量** | 单层 z | T 层 x₁, x₂, ..., x_T |
| **编码** | 一次前向 | T 步加噪 (固定) |
| **解码** | 一次前向 | T 步去噪 (学习) |
| **训练目标** | ELBO | 去噪分数匹配 |
| **重参数化** | z = μ + σ⊙ε | x_t = √ᾱ_t x₀ + √(1-ᾱ_t)ε |

### 11.3 为什么 DDPM 效果更好？

1. **多层次**: 从粗到细逐步生成
2. **简单目标**: 只预测噪声，比预测整个图像简单
3. **训练稳定**: 不像 GAN 那样容易崩溃
4. **覆盖全面**: 不易 mode collapse
