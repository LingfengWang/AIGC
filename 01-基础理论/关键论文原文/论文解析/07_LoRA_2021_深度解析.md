## 研究背景

### 1.1 大模型微调的困境

```
全参数微调的问题:

GPT-3 (175B 参数):
- 全参数微调需要: 350GB+ 显存
- 训练成本：数百万美元
- 存储成本：每个任务 700GB

问题:
- 成本太高
- 无法普及
- 只有大公司能用
```

### 1.2 参数高效微调方法

```
现有方法:

1. 前缀微调 (Prefix Tuning)
   - 添加可学习前缀
   - 但效果有限

2. Adapter
   - 插入小模块
   - 但增加推理延迟

3. 提示微调 (Prompt Tuning)
   - 只学习 prompt
   - 但效果不稳定

需求:
- 更少的参数
- 更好的效果
- 无推理延迟
```

### 1.3 动机

```
核心观察:

大模型虽然参数多，但:
- 本质维度低
- 内在秩低
- 冗余度高

洞察:
- 不需要更新所有参数
- 低秩更新就足够
- 可以大幅减少参数量
```

## 核心问题

### 2.1 核心挑战

**如何高效微调大模型，使用极少的参数达到接近全参数微调的效果？**

### 2.2 关键问题

```
问题 1: 需要多少参数？
- 全参数：100%
- Adapter: 5-10%
- LoRA: <1%

问题 2: 效果如何？
- 能否接近全参数微调？
- 能否超越提示微调？

问题 3: 推理效率？
- 是否增加延迟？
- 能否合并参数？
```

### 2.3 解决思路

```
核心思路：低秩分解

全参数更新:
W' = W + ΔW
ΔW: d×k 矩阵，参数量 d×k

LoRA 分解:
ΔW = B·A
B: d×r 矩阵
A: r×k 矩阵
r << k (通常 r=8)

参数量减少:
d×k → d×r + r×k
当 r=8, k=4096 时：
减少约 1000 倍!
```

## 核心贡献

### 3.1 低秩适应

**这是 LoRA 最核心的贡献！**

```
LoRA 的核心思想:

传统微调:
W' = W + ΔW
训练 ΔW (d×k 矩阵)

LoRA:
W' = W + B·A
训练 B (d×r) 和 A (r×k)
r << k

关键:
- 冻结 W
- 只训练 B 和 A
- 参数量大幅减少
```

### 3.2 无推理延迟

```
推理时的优势:

训练时:
output = W·x + B·(A·x)

推理时 (合并参数):
W' = W + B·A
output = W'·x

好处:
- 合并后无额外计算
- 无推理延迟
- 部署简单
```

### 3.3 通用方法

```
LoRA 可应用于:

1. Transformer
   - Attention 的 Q, K, V, O 矩阵
   - MLP 层

2. 各种任务
   - 文本分类
   - 文本生成
   - 问答
   - 翻译

3. 各种模型
   - GPT
   - BERT
   - RoBERTa
   - Stable Diffusion
```

### 3.4 实证效果

```
关键结果:

RoBERTa (Large):
- 全参数微调：100% 参数
- LoRA (r=4): 0.18% 参数
- 效果：相当甚至更好

GPT-3 (175B):
- 全参数微调：不可行
- LoRA (r=4): 0.01% 参数
- 效果：接近全参数

结论:
- 参数量减少 1000-10000 倍
- 效果接近或相同
- 可行且实用
```

## 数学原理

### 4.1 低秩分解

```
矩阵分解:

ΔW ∈ R^(d×k), rank(ΔW) = r

ΔW = B·A
B ∈ R^(d×r)
A ∈ R^(r×k)

参数量:
全参数：d×k
LoRA: d×r + r×k

当 d=4096, k=4096, r=8:
全参数：16,777,216
LoRA: 32,768 + 32,768 = 65,536
减少：256 倍!
```

### 4.2 前向传播

```
训练时:

h = Wx + ΔWx
  = Wx + BAx

其中:
- W: 冻结，不计算梯度
- B, A: 可训练
- x: 输入

实现:
1. 计算 y = Wx (冻结分支)
2. 计算 z = Ax
3. 计算 z' = Bz
4. output = y + z'
```

### 4.3 反向传播

```
梯度计算:

L = loss(output, target)

∂L/∂B = ∂L/∂output · (Ax)^T
∂L/∂A = B^T · ∂L/∂output · x^T
∂L/∂W = 0 (冻结)

关键:
- 只计算 B 和 A 的梯度
- W 不需要梯度
- 显存需求大幅减少
```

### 4.4 参数合并

```
推理时合并:

训练完成后:
W' = W + B·A

推理:
output = W'·x

好处:
- 无额外计算
- 无延迟
- 部署简单

注意:
- 合并后无法切换不同 LoRA
- 如需切换，保持分离
```

## 训练方法

### 6.1 训练算法

```
Algorithm 1: LoRA 训练

Input: 预训练模型 W, 数据集 D, 秩 r
Initialize:
  - W: 冻结
  - A: 随机初始化 (高斯)
  - B: 零初始化

repeat
    # 采样 batch
    {(x⁽ⁱ⁾, y⁽ⁱ⁾)} ~ D
    
    for each (x⁽ⁱ⁾, y⁽ⁱ⁾):
        # 前向传播
        h = Wx⁽ⁱ⁾ + B·(A·x⁽ⁱ⁾)
        output = f(h)
        
        # 计算损失
        L = loss(output, y⁽ⁱ⁾)
    
    # 反向传播 (只计算 A, B 的梯度)
    ∂L/∂A, ∂L/∂B = Backprop(L)
    
    # 更新参数
    A ← A - η ∂L/∂A
    B ← B - η ∂L/∂B
    
until 收敛
```

### 6.2 初始化策略

```
A 和 B 的初始化:

A: 高斯初始化
A ~ N(0, σ²)
σ = 1/√r 或 1/√d

B: 零初始化
B = 0

原因:
- 初始时 LoRA 输出为 0
- 不影响预训练模型
- 渐进式学习
```

### 6.3 缩放因子

```
LoRA 的缩放:

output = Wx + α·(BAx)

其中 α 是缩放因子

常用设置:
- α = r (与秩成正比)
- α = 1 (不缩放)
- α = 2r (更大缩放)

效果:
- α 太大：训练不稳定
- α 太小：学习慢
- α = r: 经验最优
```

### 6.4 训练技巧

```
1. 学习率
   - LoRA 需要更大学习率
   - 通常是全参数的 2-4 倍
   - 推荐：1e-4 到 5e-4

2. 秩选择
   - 简单任务：r=4-8
   - 复杂任务：r=16-64
   - 默认：r=8

3. 目标模块
   - Attention: Q, V 最重要
   - 全部 Attention: 效果更好
   - 加 MLP: 边际收益小
```

## 代码实现

### 8.1 LoRA 层

```python
class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=8, alpha=None):
        super().__init__()
        
        # 原始权重 (冻结)
        self.linear = nn.Linear(in_features, out_features)
        self.linear.requires_grad_(False)
        
        # LoRA 参数
        self.r = r
        self.alpha = alpha if alpha else r
        self.scaling = self.alpha / self.r
        
        # A: 高斯初始化
        self.lora_A = nn.Parameter(
            torch.randn(r, in_features) * (1 / r)
        )
        
        # B: 零初始化
        self.lora_B = nn.Parameter(
            torch.zeros(out_features, r)
        )
    
    def forward(self, x):
        # 原始分支 (冻结)
        original = self.linear(x)
        
        # LoRA 分支
        lora = torch.matmul(x, self.lora_A.T)
        lora = torch.matmul(lora, self.lora_B.T)
        lora = lora * self.scaling
        
        return original + lora
```

### 8.2 LoRA for Attention

```python
class LoRAAttention(nn.Module):
    def __init__(self, dim, num_heads, r=8):
        super().__init__()
        
        self.q_proj = LoRALinear(dim, dim, r=r)
        self.k_proj = LoRALinear(dim, dim, r=r)
        self.v_proj = LoRALinear(dim, dim, r=r)
        self.o_proj = LoRALinear(dim, dim, r=r)
        
        self.num_heads = num_heads
        self.head_dim = dim // num_heads
    
    def forward(self, x):
        B, N, C = x.shape
        
        # Q, K, V (带 LoRA)
        Q = self.q_proj(x).reshape(B, N, self.num_heads, self.head_dim).transpose(1, 2)
        K = self.k_proj(x).reshape(B, N, self.num_heads, self.head_dim).transpose(1, 2)
        V = self.v_proj(x).reshape(B, N, self.num_heads, self.head_dim).transpose(1, 2)
        
        # Attention
        attn = (Q @ K.transpose(-2, -1)) / (self.head_dim ** 0.5)
        attn = attn.softmax(dim=-1)
        
        # Output
        out = (attn @ V).transpose(1, 2).reshape(B, N, C)
        out = self.o_proj(out)
        
        return out
```

### 8.3 训练代码

```python
def train_lora(model, dataloader, optimizer, device, epochs=10):
    # 冻结所有参数
    for param in model.parameters():
        param.requires_grad = False
    
    # 只开启 LoRA 参数
    for name, module in model.named_modules():
        if isinstance(module, LoRALinear):
            module.lora_A.requires_grad = True
            module.lora_B.requires_grad = True
    
    model.train()
    
    for epoch in range(epochs):
        total_loss = 0
        
        for batch_idx, (inputs, targets) in enumerate(dataloader):
            inputs, targets = inputs.to(device), targets.to(device)
            optimizer.zero_grad()
            
            # 前向传播
            outputs = model(inputs)
            
            # 计算损失
            loss = F.cross_entropy(outputs, targets)
            
            # 反向传播
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
        
        avg_loss = total_loss / len(dataloader)
        print(f'Epoch {epoch+1}: Loss = {avg_loss:.4f}')
```

### 8.4 参数合并

```python
@torch.no_grad()
def merge_lora(model):
    """合并 LoRA 参数到原始权重"""
    
    for name, module in model.named_modules():
        if isinstance(module, LoRALinear):
            # 获取原始权重
            W = module.linear.weight.data
            
            # 计算 LoRA 更新
            BA = module.lora_B.T @ module.lora_A
            BA = BA * module.scaling
            
            # 合并
            W_merged = W + BA
            
            # 替换
            module.linear.weight.data = W_merged
            
            # 移除 LoRA
            del module.lora_A
            del module.lora_B
```

## 深入理解

### 9.1 为什么低秩有效？

```
直觉解释 1: 内在维度

大模型虽然参数多，但:
- 有效自由度少
- 内在维度低
- 冗余度高

证据:
- 剪枝研究：90% 参数可剪
- 量化研究：4bit 效果接近
- LoRA: 低秩更新足够

结论:
- 不需要全参数更新
- 低秩更新捕捉主要变化
```

```
直觉解释 2: 任务特异性

预训练模型已经学会:
- 通用语言知识
- 世界知识
- 推理能力

微调只需要:
- 适配特定任务
- 小的调整
- 低秩更新足够
```

### 9.2 零初始化的重要性

```
为什么 B 要零初始化？

如果随机初始化:
- 初始 LoRA 输出随机
- 破坏预训练模型
- 训练不稳定

零初始化:
- 初始 LoRA 输出为 0
- 保持预训练能力
- 渐进式学习
- 训练稳定

类比:
- 类似 ResNet 的残差
- 初始为恒等映射
- 学习增量
```

### 9.3 秩的选择

```
秩 r 的选择:

小 r (r=4-8):
- 参数量极少
- 适合简单任务
- 训练快

中 r (r=16-32):
- 平衡参数和效果
- 适合大多数任务
- 推荐默认

大 r (r=64-256):
- 参数量较多
- 适合复杂任务
- 边际收益递减

经验法则:
- 从 r=8 开始
- 效果不好再增大
- 很少需要 r>64
```

### 9.4 与 Adapter 的对比

```
LoRA vs Adapter:

| 特性 | LoRA | Adapter |
|------|------|---------|
| **参数量** | 0.01-0.1% | 3-10% |
| **推理延迟** | 无 (可合并) | 有 |
| **实现难度** | 简单 | 中等 |
| **效果** | 好 | 好 |
| **灵活性** | 高 | 中 |

结论:
- LoRA 参数更少
- LoRA 无推理延迟
- LoRA 更简单
- LoRA 成为主流
```
