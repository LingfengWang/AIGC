## 研究背景

### 1.1 DDPM 的困境

DDPM (2020) 虽然成功，但存在严重问题：

| 问题 | 表现 | 影响 |
|------|------|------|
| **计算量大** | 在像素空间训练 | 需要数百 GPU 天 |
| **分辨率限制** | 只能训练 64×64 或 128×128 | 无法生成高清图像 |
| **推理慢** | 1000 步去噪 | 生成一张图需要几分钟 |

### 1.2 高分辨率挑战

```
像素空间 Diffusion 的计算量:

64×64×3 图像:
- 维度：12,288
- 计算量：可接受

256×256×3 图像:
- 维度：196,608
- 计算量：16 倍增长

512×512×3 图像:
- 维度：786,432
- 计算量：64 倍增长！

问题:
- 显存不够
- 训练太慢
- 无法实用
```

### 1.3 隐空间的启发

```
VAE 的启示:

VAE 可以将图像压缩到隐空间:
512×512×3 → 64×64×4

压缩比: 48 倍!

洞察:
如果在隐空间做 Diffusion 会怎样？
- 计算量减少 48 倍
- 可以训练更高分辨率
- 推理也更快
```

## 核心贡献

### 3.1 隐空间扩散

**这是 LDM 最核心的贡献！**

```
传统 Diffusion (像素空间):
x₀ (512×512×3) → 加噪 → 去噪 → x₀'

LDM (隐空间):
x₀ → VAE Encoder → z (64×64×4)
z → 加噪 → 去噪 → z'
z' → VAE Decoder → x₀'

计算量对比:
像素空间：512×512×3 = 786,432 维
隐空间：64×64×4 = 16,384 维
压缩比：48 倍!
```

### 3.2 Cross-Attention 条件机制

```
通用条件控制框架:

条件 y (文本、图像、布局等)
  ↓
条件编码器 τ_θ(y)
  ↓
Cross-Attention
  ↓
Diffusion UNet

关键:
- 统一的接口
- 支持多种条件类型
- 灵活可扩展
```

### 3.3 两阶段训练

```
阶段 1: 训练 VAE

输入：大量图像
目标：学习压缩和解压缩
输出：VAE Encoder 和 Decoder
状态：训练完成后固定参数

阶段 2: 训练 Diffusion

输入：VAE 压缩的隐变量
目标：学习去噪
条件：Cross-Attention 注入
输出：条件图像生成模型
```

## 数学原理

### 4.1 VAE 基础

```
VAE 的编码 - 解码:

编码器: q(z|x) = N(z; μ(x), σ²(x)I)
解码器: p(x|z)

训练目标:
L_VAE = E[log p(x|z)] - KL(q(z|x) || p(z))

其中:
- x: 原始图像
- z: 隐变量
- KL 散度：正则化隐空间
```

### 4.2 隐空间 Diffusion

```
前向过程 (在隐空间):
q(z_t | z_0) = N(z_t; √ᾱ_t z_0, (1-ᾱ_t)I)

反向过程:
p_θ(z_{t-1} | z_t, y) = N(z_{t-1}; μ_θ(z_t, t, y), Σ_θ(z_t, t))

损失函数:
L = E_{z_0,ε,t,y}[||ε - ε_θ(z_t, t, y)||²]

其中:
- z_0: VAE 编码的隐变量
- y: 条件 (文本、图像等)
- ε_θ: UNet 预测的噪声
```

### 4.3 Cross-Attention

```
Cross-Attention 机制:

Q = W_Q · φ(z_t)  (来自 Diffusion 特征)
K = W_K · τ_θ(y)  (来自条件编码)
V = W_V · τ_θ(y)  (来自条件编码)

Attention(Q, K, V) = softmax(QK^T / √d) V

其中:
- φ(z_t): Diffusion UNet 的中间特征
- τ_θ(y): 条件编码器输出 (如 CLIP 文本嵌入)
- W_Q, W_K, W_V: 可学习投影矩阵
```

### 4.4 感知压缩

```
VAE 的设计原则:

目标：在压缩率和重建质量之间平衡

压缩率:
- 输入：512×512×3
- 输出：64×64×4
- 压缩：48 倍

重建质量:
- 使用感知损失 (Perceptual Loss)
- 使用 LPIPS 损失
- 保持高频细节

关键:
- 不能压缩太多 (丢失信息)
- 不能压缩太少 (计算量大)
- 48 倍是经验最优
```

└─────────────────────────────────────────────────────────────┘

### 5.2 VAE 架构

```
VAE Encoder:
输入：512×512×3 图像

Downsample × 3:
- 512×512 → 256×256 → 128×128 → 64×64
- 每层：Conv + ResBlock

输出：μ, logσ² (各 64×64×4)

VAE Decoder:
输入：z (64×64×4)

Upsample × 3:
- 64×64 → 128×128 → 256×256 → 512×512
- 每层：ConvTranspose + ResBlock

输出：512×512×3 图像
```

### 5.3 UNet 架构

```
Diffusion UNet:

输入：z_t (64×64×4) + t (时间步) + y (条件)

Downsample 块 (4 层):
- 64×64 → 32×32 → 16×16 → 8×8
- 每层：ResBlock + CrossAttention

Middle 块:
- 8×8 分辨率
- ResBlock + Attention

Upsample 块 (4 层):
- 8×8 → 16×16 → 32×32 → 64×64
- 每层：ResBlock + CrossAttention + Skip Connection

输出：噪声 ε_θ
```

### 5.4 条件编码器

```
文本条件:
使用 CLIP Text Encoder
- 输入：文本 prompt (最多 77 tokens)
- 输出：77×768 文本嵌入
- 预训练，固定参数

图像条件:
使用 VAE Encoder 或其他编码器
- 输入：条件图像
- 输出：隐变量或特征

布局条件:
使用简单的 MLP 或 CNN
- 输入：布局图
- 输出：条件特征
```

## 训练方法

### 6.1 两阶段训练

```
Algorithm 1: LDM 训练

阶段 1: 训练 VAE
Input: 图像数据集 {x⁽¹⁾, ..., x⁽ᴺ⁾}
Initialize: VAE 参数 φ, ψ

repeat
    for each x⁽ⁱ⁾:
        # 编码
        μ, σ² = Encoder_φ(x⁽ⁱ⁾)
        z = μ + σ ⊙ ε
        
        # 解码
        x' = Decoder_ψ(z)
        
        # 损失
        L = 重构损失 + KL 损失 + 感知损失
    
    更新 φ, ψ
until 收敛

固定 VAE 参数

阶段 2: 训练 Diffusion
Input: 图像 + 条件数据集 {(x⁽¹⁾, y⁽¹⁾), ...}
Initialize: UNet 参数 θ

repeat
    for each (x⁽ⁱ⁾, y⁽ⁱ⁾):
        # VAE 编码 (固定)
        z₀ = Encoder(x⁽ⁱ⁾)
        
        # 随机时间步和噪声
        t ~ Uniform(1, T)
        ε ~ N(0, I)
        
        # 加噪
        z_t = √ᾱ_t z₀ + √(1-ᾱ_t) ε
        
        # 预测噪声 (带条件)
        ε_θ = UNet(z_t, t, y⁽ⁱ⁾)
        
        # 损失
        L = ||ε - ε_θ||²
    
    更新 θ
until 收敛
```

### 6.2 条件训练技巧

```
Classifier-Free Guidance:

训练时:
- 以概率 p 丢弃条件 y (用空条件替代)
- 模型学会无条件生成

推理时:
ε_guided = ε_uncond + s × (ε_cond - ε_uncond)

其中 s 是 guidance scale
- s = 1: 无条件
- s = 7.5: 默认值
- s 越大：越符合条件，但多样性降低
```

### 6.3 数据增强

```
训练数据增强:

1. 随机裁剪
   - 从大图中随机裁剪 512×512
   - 增加数据多样性

2. 随机翻转
   - 水平翻转
   - 增加对称不变性

3. 颜色抖动
   - 亮度、对比度、饱和度
   - 增强颜色鲁棒性
```

## 代码实现

### 8.1 VAE 模型

```python
class VAE(nn.Module):
    def __init__(self, in_channels=3, latent_channels=4):
        super().__init__()
        
        # Encoder
        self.encoder = nn.Sequential(
            nn.Conv2d(in_channels, 128, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.Conv2d(128, 256, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.Conv2d(256, 512, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.Conv2d(512, latent_channels * 2, 3, padding=1),
        )
        
        # Decoder
        self.decoder = nn.Sequential(
            nn.Conv2d(latent_channels, 512, 3, padding=1),
            nn.ReLU(),
            nn.ConvTranspose2d(512, 256, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.ConvTranspose2d(256, 128, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.ConvTranspose2d(128, in_channels, 3, stride=2, padding=1),
            nn.Sigmoid(),
        )
    
    def encode(self, x):
        h = self.encoder(x)
        mu, logvar = h.chunk(2, dim=1)
        return mu, logvar
    
    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)
        return mu + eps * std
    
    def decode(self, z):
        return self.decoder(z)
```

    # 阶段 1: 训练 VAE (略)
    
    # 阶段 2: 训练 Diffusion
    vae.eval()
    vae.requires_grad_(False)
    unet.train()
    
    for epoch in range(epochs):
        total_loss = 0
        
        for batch_idx, (images, captions) in enumerate(dataloader):
            images = images.to(device)
            
            # VAE 编码 (固定)
            with torch.no_grad():
                mu, logvar = vae.encode(images)
                z0 = vae.reparameterize(mu, logvar)
            
            # 随机时间步和噪声
            t = torch.randint(0, 1000, (images.shape[0],)).to(device)
            noise = torch.randn_like(z0)
            
            # 加噪
            zt = add_noise(z0, noise, t)
            
            # 文本编码
            cond = clip_encode(captions)
            
            # 预测噪声
            noise_pred = unet(zt, t, cond)
            
            # 损失
            loss = F.mse_loss(noise_pred, noise)
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
        
        avg_loss = total_loss / len(dataloader)
        print(f'Epoch {epoch+1}: Loss = {avg_loss:.4f}')

### 8.4 采样函数

```python
@torch.no_grad()
def sample_ldm(vae, unet, prompt, device='cuda', num_steps=50):
    vae.eval()
    unet.eval()
    
    # 文本编码
    cond = clip_encode(prompt)
    uncond = clip_encode("")  # 无条件
    
    # 从纯噪声开始
    z = torch.randn(1, 4, 64, 64).to(device)
    
    # DDIM 采样
    scheduler = DDIMScheduler(num_steps)
    
    for t in scheduler.timesteps:
        # 预测噪声
        noise_pred_cond = unet(z, t, cond)
        noise_pred_uncond = unet(z, t, uncond)
        
        # Classifier-Free Guidance
        noise_pred = uncond + 7.5 * (noise_pred_cond - noise_pred_uncond)
        
        # DDIM 采样一步
        z = scheduler.step(noise_pred, t, z)
    
    # VAE 解码
    image = vae.decode(z)
    
    return image
```

## 深入理解

### 9.1 为什么隐空间有效？

```
直觉解释 1: 信息密度

像素空间:
- 相邻像素高度相关
- 大量冗余信息
- 有效信息密度低

隐空间:
- VAE 已经去除冗余
- 信息更紧凑
- 有效信息密度高

结果:
- 在隐空间学习更高效
- 需要的计算量更少
```

```
直觉解释 2: 感知质量

VAE 的设计:
- 使用感知损失
- 保持高频细节
- 人眼难以察觉差异

结果:
- 虽然是有损压缩
- 但感知质量高
- 适合图像生成
```

### 9.2 Cross-Attention 的优势

```
传统条件注入:
条件 y → 拼接 → UNet 输入

问题:
- 条件信息被稀释
- 难以捕捉长程依赖

Cross-Attention:
条件 y → K, V
UNet 特征 → Q
Attention(Q, K, V) → 条件特征

优势:
- 显式建模条件 - 特征关系
- 可以捕捉长程依赖
- 灵活支持多种条件
```

### 9.3 Classifier-Free Guidance

```
为什么有效？

训练时:
- 模型同时学习有条件和无条件生成
- 学会区分两种模式

推理时:
ε_guided = ε_uncond + s × (ε_cond - ε_uncond)

解释:
- ε_uncond: 无条件生成方向
- ε_cond - ε_uncond: 条件引导方向
- s: 引导强度

效果:
- s 越大：越符合条件
- 但多样性降低
- s=7.5 是经验最优
```

### 9.4 隐空间的几何结构

```
VAE 隐空间的特点:

1. 连续性
   - 相近的 z 产生相近的 x
   - 支持插值

2. 完整性
   - 从隐空间采样都能产生合理图像
   - 支持生成

3. 解耦性 (理想情况)
   - 不同维度控制不同语义
   - 但实际解耦不完全

应用:
- 隐空间插值
- 隐空间编辑
- 图像混合
```

## 与 DDPM 的关系

### 10.1 理论关系

```
LDM = VAE + DDPM

VAE:
- 压缩：x → z
- 解压缩：z → x

DDPM:
- 在 z 空间进行
- 学习 z 的生成

关系:
- DDPM 是 LDM 的子组件
- LDM 是 DDPM 的扩展
```

### 10.2 架构对比

| 特性 | DDPM | LDM |
|------|------|-----|
