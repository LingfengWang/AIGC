## 1. VAE (2013)

**论文**: Auto-Encoding Variational Bayes  
**作者**: Diederik P. Kingma, Max Welling  
**机构**: University of Amsterdam  
**发表**: ICLR 2014  
**引用**: 25,000+  
**文件**: `05_VAE_2013_Kingma.pdf` (3.7 MB)

### 🎯 核心问题

如何学习数据的**连续隐变量表示** (continuous latent representation)，并能够从中生成新样本？

在 VAE 之前，生成模型主要有：
- GAN：训练不稳定，mode collapse
- 自回归模型：生成慢

### 💡 核心贡献

1. **重参数化技巧 (Reparameterization Trick)**
   ```
   传统采样：z ~ N(μ, σ²)  ← 不可导，无法反向传播
   重参数化：ε ~ N(0, 1), z = μ + σ·ε  ← 可导！
   ```

2. **ELBO 损失函数**
   ```
   L(θ,φ; x) = E_q[log p(x|z)] - KL(q(z|x) || p(z))
              重构损失           正则化项
   
   重构损失：让解码器能重建原图
   正则化项：让隐变量分布接近标准正态分布
   ```

3. **端到端训练**
   - 编码器 q(z|x)：图像 → 隐变量分布
   - 解码器 p(x|z)：隐变量 → 图像
   - 可以联合训练

### 🏗️ 模型架构

```
输入图像 x (3×256×256)
    ↓
编码器 (CNN)
    ↓
μ, logσ² (隐变量分布参数)
    ↓
重参数化：z = μ + σ·ε, ε~N(0,1)
    ↓
解码器 (CNN)
    ↓
重建图像 x' (3×256×256)
```

### 📐 核心公式

**1. 前向过程**
```
编码器：q_φ(z|x) = N(z; μ_φ(x), diag(σ²_φ(x)))
解码器：p_θ(x|z) = N(x; μ_θ(z), diag(σ²_θ(z)))
```

**2. 重参数化**
```
z = μ_φ(x) + σ_φ(x) ⊙ ε,  其中 ε ~ N(0, I)
```

**3. ELBO 损失**
```
L = E_{x~D}[E_{z~q_φ(·|x)}[-log p_θ(x|z)] + KL(q_φ(z|x) || p(z))]
```

**4. KL 散度解析解** (当先验 p(z)=N(0,I) 时)
```
KL(q_φ(z|x) || p(z)) = -½ Σ(1 + log σ²_i - μ²_i - σ²_i)
```

### 🤔 为什么重要

1. **生成模型基础**: VAE 是现代生成模型的基石
2. **隐空间概念**: Diffusion 可以看作"多个 VAE 串联"
3. **重参数化**: 这个技巧被广泛使用
4. **理论优美**: 基于变分推断，有坚实数学基础

### 🔍 深入理解

**Q1: 为什么需要重参数化？**

因为我们需要对 z 采样，但采样操作不可导。重参数化把随机性移到 ε 上，而 ε 不需要梯度，μ 和 σ 可以正常反向传播。

**Q2: KL 散度的作用是什么？**

KL 散度是正则化项，防止模型"作弊"。如果没有 KL 项，编码器可能给每个样本分配完全不同的隐变量，导致无法生成新样本。

**Q3: VAE 和 Diffusion 的关系？**

可以这样理解：
- VAE: 编码一次 → 解码一次
- Diffusion: 编码 T 次 (加噪) → 解码 T 次 (去噪)

Diffusion 是"多层次"的 VAE。

### 📝 代码示例

```python
import torch
import torch.nn as nn

class VAE(nn.Module):
    def __init__(self):
        super().__init__()
        # 编码器
        self.encoder = nn.Sequential(
            nn.Conv2d(3, 32, 3, stride=2),
            nn.ReLU(),
            nn.Conv2d(32, 64, 3, stride=2),
            nn.ReLU(),
            nn.Flatten(),
        )
        self.fc_mu = nn.Linear(64*64*64, 128)
        self.fc_logvar = nn.Linear(64*64*64, 128)
        
        # 解码器
        self.decoder = nn.Sequential(
            nn.Linear(128, 64*64*64),
            nn.Unflatten(1, (64, 64, 64)),
            nn.ConvTranspose2d(64, 32, 3, stride=2),
            nn.ReLU(),
            nn.ConvTranspose2d(32, 3, 3, stride=2),
            nn.Sigmoid(),
        )
    
    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)
        return mu + eps * std
    
    def forward(self, x):
        h = self.encoder(x)
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)
        z = self.reparameterize(mu, logvar)
        x_recon = self.decoder(z)
        return x_recon, mu, logvar
    
    def loss(self, x, x_recon, mu, logvar):
        # 重构损失
        recon_loss = nn.functional.mse_loss(x_recon, x, reduction='sum')
        # KL 损失
        kl_loss = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
        return recon_loss + kl_loss
```

## 2. DDPM (2020)

**论文**: Denoising Diffusion Probabilistic Models  
**作者**: Jonathan Ho, Ajay Jain, Pieter Abbeel  
**机构**: UC Berkeley  
**发表**: NeurIPS 2020  
**引用**: 15,000+  
**文件**: `01_DDPM_2020_Ho.pdf` (9.8 MB)

### 🎯 核心问题

如何生成高质量图像，同时避免 GAN 的训练不稳定和 mode collapse 问题？

### 💡 核心贡献

1. **扩散概率模型框架**
   - 前向过程：逐步加噪 (固定)
   - 反向过程：逐步去噪 (学习)

2. **预测噪声而非原图**
   ```
   损失函数：L = E[||ε - ε_θ(x_t, t)||²]
   预测目标：噪声 ε，而不是原图 x_0
   ```
   这个反直觉的设计是关键！

3. **任意时刻直接采样**
   ```
   q(x_t | x_0) = N(x_t; √ᾱ_t x_0, (1-ᾱ_t)I)
   可以直接从 x_0 计算任意 t 的 x_t，无需一步步加噪
   ```

### 🏗️ 模型架构

```
训练过程:
1. 从数据集采样 x_0
2. 随机采样时间步 t ~ Uniform(1, T)
3. 采样噪声 ε ~ N(0, I)
4. 计算 x_t = √ᾱ_t x_0 + √(1-ᾱ_t) ε
5. 训练网络预测 ε_θ(x_t, t)
6. 最小化 ||ε - ε_θ(x_t, t)||²

采样过程:
1. 从纯噪声 x_T ~ N(0, I) 开始
2. 对于 t = T, T-1, ..., 1:
   x_{t-1} = 1/√α_t (x_t - (1-α_t)/√(1-ᾱ_t) ε_θ(x_t, t)) + σ_t z
3. 返回 x_0
```

### 📐 核心公式

**1. 前向扩散过程**
```
q(x_t | x_{t-1}) = N(x_t; √(1-β_t)x_{t-1}, β_t I)
q(x_t | x_0) = N(x_t; √ᾱ_t x_0, (1-ᾱ_t)I)

其中：α_t = 1 - β_t, ᾱ_t = ∏_{s=1}^t α_s
```

**2. 反向去噪过程**
```
p_θ(x_{t-1} | x_t) = N(x_{t-1}; μ_θ(x_t, t), Σ_θ(x_t, t))
```

**3. 简化损失函数**
```
L_simple = E_{t,x_0,ε}[||ε - ε_θ(x_t, t)||²]
```

**4. 噪声调度**
```
β_t 从 10⁻⁴ 线性增加到 0.02
```

### 🤔 为什么重要

1. **Diffusion 奠基**: 开启了 Diffusion 时代
2. **训练稳定**: 不像 GAN 那样容易崩溃
3. **生成质量高**: 媲美当时最好的 GAN
4. **理论优美**: 基于变分推断

### 🔍 深入理解

**Q1: 为什么预测噪声而不是原图？**

实验发现预测噪声效果更好。直觉解释：
- 噪声 ε 是简单的高斯分布
- 原图 x_0 是复杂的数据分布
- 学习简单分布更容易

**Q2: 为什么需要 1000 步？**

每一步只去除一点噪声，步数越多，生成质量越好。但这也导致采样慢（后来 DDIM 解决了这个问题）。

**Q3: 噪声调度 β_t 如何选择？**

线性调度从 10⁻⁴ 到 0.02 是经验选择。前期加噪少，后期加噪多，这样既能保留信息，又能充分混合。

### 📝 代码示例

```python
import torch
import torch.nn as nn

class DDPM:
    def __init__(self, T=1000, beta_start=1e-4, beta_end=0.02):
        self.T = T
        self.betas = torch.linspace(beta_start, beta_end, T)
        self.alphas = 1. - self.betas
        self.alphas_cumprod = torch.cumprod(self.alphas, dim=0)
    
    def add_noise(self, x0, noise, t):
        """一次性添加 t 步的噪声"""
        sqrt_alpha_cumprod = torch.sqrt(self.alphas_cumprod[t])
        sqrt_one_minus_alpha = torch.sqrt(1 - self.alphas_cumprod[t])
        return sqrt_alpha_cumprod * x0 + sqrt_one_minus_alpha * noise
    
    def get_loss(self, model, x0):
        """计算训练损失"""
        batch_size = x0.shape[0]
        # 随机采样时间步
        t = torch.randint(0, self.T, (batch_size,))
        # 随机噪声
        noise = torch.randn_like(x0)
        # 加噪
        xt = self.add_noise(x0, noise, t)
        # 预测噪声
        noise_pred = model(xt, t)
        # MSE 损失
        return nn.functional.mse_loss(noise_pred, noise)
    
    @torch.no_grad()
    def sample(self, model, shape=(1, 3, 32, 32)):
        """采样"""
        model.eval()
        x = torch.randn(shape)  # 从纯噪声开始
        
        for t in reversed(range(self.T)):
            t_tensor = torch.tensor([t]).expand(shape[0])
            noise_pred = model(x, t_tensor)
            
            # 计算均值
            alpha = self.alphas[t]
            alpha_cumprod = self.alphas_cumprod[t]
            beta = self.betas[t]
            
            x = (x - beta / torch.sqrt(1 - alpha_cumprod) * noise_pred) / torch.sqrt(alpha)
            
            # 添加噪声 (除了最后一步)
            if t > 0:
                z = torch.randn_like(x)
                x += torch.sqrt(beta) * z
        
        return x
```

## 3. DDIM (2020)

**论文**: Denoising Diffusion Implicit Models  
**作者**: Jiaming Song et al.  
**机构**: Stanford, UC Berkeley  
**发表**: ICLR 2021  
**引用**: 8,000+  
**文件**: `02_DDIM_2020_Song.pdf` (10 MB)

### 🎯 核心问题

DDPM 需要 1000 步采样，太慢了！能否减少采样步数？

### 💡 核心贡献

1. **非马尔可夫扩散过程**
   ```
   DDPM: x_t → x_{t-1} → ... → x_0  (马尔可夫)
   DDIM: x_t → x_{t-Δ} → ... → x_0  (可以跳步！)
   ```

2. **确定性采样**
   - DDPM 是随机过程
   - DDIM 是确定性的
   - 相同起点 → 相同结果

3. **统一框架**
   ```
   DDIM 包含 DDPM 作为特例
   通过调节 η 参数，可以在两者之间插值
   ```

### 📐 核心公式

**1. DDIM 采样**
```
x_{t-1} = √ᾱ_{t-1} (x_t - √(1-ᾱ_t) ε_θ(x_t)) / √ᾱ_t
        + √(1-ᾱ_{t-1} - σ_t²) ε_θ(x_t)
        + σ_t z,  其中 z ~ N(0, I)
```

**2. 噪声调度**
```
当 σ_t = 0 时：确定性采样 (DDIM)
当 σ_t = √((1-ᾱ_{t-1})/(1-ᾱ_t)) √(1-ᾱ_t/ᾱ_{t-1}) β_t 时：随机采样 (DDPM)
```

### 🤔 为什么重要

1. **加速采样**: 从 1000 步减少到 50-100 步
2. **确定性**: 便于调试和复现
3. **理论贡献**: 统一了扩散模型框架

## 4. Score SDE (2020)

**论文**: Score-Based Generative Modeling through Stochastic Differential Equations  
**作者**: Yang Song et al.  
**机构**: Stanford, Google Brain  
**发表**: ICLR 2021  
**引用**: 5,000+  
**文件**: `16_ScoreSDE_2020_Song.pdf` (26 MB)

### 🎯 核心问题

如何统一 Score Matching 和 Diffusion Models 的理论框架？

### 💡 核心贡献

1. **分数匹配 (Score Matching)**
   ```
   学习目标：s_θ(x) ≈ ∇_x log p(x)
   分数函数：数据分布的对数梯度
   ```

2. **随机微分方程 (SDE) 框架**
   ```
   前向 SDE: dx = f(x,t)dt + g(t)dw
   反向 SDE: dx = [f(x,t) - g(t)² ∇_x log p_t(x)]dt + g(t)dw̄
   ```

3. **概率流 ODE**
   ```
   确定性的连续时间生成过程
   可以用于精确似然计算
   ```

### 🤔 为什么重要

1. **理论统一**: 统一了多个生成模型框架
2. **SDE 视角**: 提供了新的理论工具
3. **后续影响**: 很多工作基于此框架

## 6. LDM/Stable Diffusion (2021)

**论文**: High-Resolution Image Synthesis with Latent Diffusion Models  
**作者**: Robin Rombach et al.  
**机构**: CompVis (LMU Munich), Stability AI  
**发表**: CVPR 2022  
**引用**: 10,000+  
**文件**: `03_LDM_2021_Rombach.pdf` (37 MB)

### 🎯 核心问题

DDPM 在像素空间训练，计算量太大。如何生成高分辨率图像？

### 💡 核心贡献

1. **隐空间扩散**
   ```
   像素空间：512×512×3 = 786,432 维
   隐空间：64×64×4 = 16,384 维  ← 压缩 48 倍!
   ```

2. **两阶段训练**
   - 阶段 1: 训练 VAE，学习压缩
   - 阶段 2: 在隐空间训练 Diffusion

3. **Cross-Attention 条件机制**
   ```
   文本 → CLIP Text Encoder → 文本嵌入
   图像隐变量 + 文本嵌入 → Cross-Attention → 条件生成
   ```

### 🏗️ 模型架构

```
训练:
图像 → VAE Encoder → 隐变量 z → Diffusion UNet → 隐变量 z' → VAE Decoder → 重建图像

采样:
文本 → CLIP → 文本嵌入
噪声 → Diffusion UNet (条件：文本嵌入) → 隐变量 → VAE Decoder → 图像
```

### 📐 核心公式

**1. 隐变量压缩**
```
z = E(x),  其中 E 是 VAE 编码器
x' = D(z), 其中 D 是 VAE 解码器
```

**2. Cross-Attention**
```
Q = W_Q · φ_θ(z_t),  K = W_K · τ_θ(y),  V = W_V · τ_θ(y)
Attention(Q,K,V) = softmax(QK^T/√d) V

其中：z_t 是隐变量，y 是文本，τ_θ 是文本编码器
```

### 🤔 为什么重要

1. **计算效率**: 训练成本降低 48 倍
2. **开源生态**: 引爆社区创新
3. **商业化**: AIGC 进入大众视野

## 7. ControlNet (2023)

**论文**: Adding Conditional Control to Text-to-Image Diffusion Models  
**作者**: Lvmin Zhang, Maneesh Agrawala  
**机构**: Stanford  
**发表**: ICCV 2023  
**引用**: 3,000+  
**文件**: `06_ControlNet_2023_Zhang.pdf` (16 MB)

### 🎯 核心问题

SD 只能文本控制，无法精确控制构图。如何用边缘图、深度图、姿态图等控制生成？

### 💡 核心贡献

1. **锁定 + 复制策略**
   ```
   锁定原始 SD 模型参数 (不更新)
   复制一份 encoder 层，训练条件控制
   ```

2. **Zero Convolution**
   ```
   初始化为零卷积：输出全为 0
   ```

## 9. LoRA (2021)

**论文**: LoRA: Low-Rank Adaptation of Large Language Models  
**作者**: Edward J. Hu et al.  
**机构**: Microsoft  
**发表**: ICLR 2022  
**引用**: 5,000+  
**文件**: `07_LoRA_2021_Hu.pdf` (1.5 MB)

### 🎯 核心问题

大模型全参数微调成本太高。能否只微调少量参数？

### 💡 核心贡献

1. **低秩分解**
   ```
   原始权重更新：ΔW (d×k，参数量大)
   LoRA 分解：ΔW = B·A  (B: d×r, A: r×k, r << d)
   r=8 时，参数量减少 1000 倍!
   ```

2. **冻结 + 旁路**
   ```
   冻结原始模型权重 W
   训练 LoRA 权重 A, B
   推理时：W' = W + B·A
   ```

3. **无推理延迟**
   ```
   训练完成后可合并：W' = W + B·A
   推理时无额外计算开销
   ```

### 🤔 为什么重要

1. **高效微调**: 参数量减少 1000 倍
2. **2023 年应用于 SD**: 催生数十万 LoRA 模型
3. **广泛应用**: NLP、CV、多模态

## 13. Rectified Flow (2022)

**论文**: Flow Straight and Fast: Learning to Generate and Transfer Data with Rectified Flow  
**作者**: Xingchao Liu et al.  
**机构**: UT Austin  
**发表**: ICLR 2023  
**引用**: 1,000+  
**文件**: `08_RectifiedFlow_2022_Liu.pdf` (17 MB)

### 🎯 核心问题

Diffusion 的噪声路径是弯曲的，采样效率低。能否让生成路径更直、更快？

### 💡 核心贡献

1. **直线路径**
   ```
   Diffusion: x_0 → 弯曲路径 → x_1
   Rectified Flow: x_0 → 直线路径 → x_1
   ```

2. **常微分方程 (ODE)**
   ```
   dx/dt = v(x, t)  ← 学习速度场
   从噪声积分到数据：x_1 = x_0 + ∫v(x,t)dt
   ```

3. **更少采样步数**
   ```
   Diffusion: 100-1000 步
   Rectified Flow: 10-50 步即可
   ```

### 🤔 为什么重要

1. **效率**: 采样速度快 10-20 倍
2. **质量**: 生成质量相当或更好
3. **采用**: SD3、FLUX 都采用
