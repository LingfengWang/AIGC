## 研究背景

### 1.1 UNet 的统治地位

```
2020-2022 年 Diffusion 架构:

DDPM → UNet
DDIM → UNet
LDM/SD → UNet
ControlNet → UNet

UNet 的特点:
- CNN 架构
- 归纳偏置：局部性、平移不变性
- 在图像任务上效果好

问题:
- 扩展性有限
- 难以利用大规模数据
- 架构不够通用
```

### 1.2 Transformer 的成功

```
NLP 领域:
- Transformer 完全统治
- GPT、BERT 等都是 Transformer
- 可扩展性极好

CV 领域:
- ViT (Vision Transformer) 证明可行性
- 在分类任务上超越 CNN
- 但生成任务还是 UNet

关键问题:
Transformer 能否用于 Diffusion 生成？
```

### 1.3 动机

```
核心动机:

1. 架构统一
   - NLP 和 CV 使用相同架构
   - 便于多模态融合

2. 可扩展性
   - Transformer 已被证明可扩展
   - 更大模型 = 更好效果

3. 长程依赖
   - Attention 捕捉全局信息
   - UNet 的卷积只有局部感受野
```

## 核心问题

### 2.1 核心挑战

**能否用纯 Transformer 架构替代 UNet，实现可扩展的 Diffusion 模型？**

### 2.2 关键问题

```
问题 1: Transformer 能处理图像吗？
- 图像是 2D 结构
- Transformer 处理 1D 序列
- 如何转换？

问题 2: 性能能超越 UNet 吗？
- UNet 有归纳偏置
- Transformer 没有
- 需要更多数据？

问题 3: 扩展性如何？
- 能否训练更大模型？
- 性能随规模如何变化？
```

### 2.3 解决思路

```
核心思路：Patchify + Transformer

1. Patchify
   - 将图像切成 patches
   - 类似 ViT 的做法
   - 2D → 1D 序列

2. Transformer
   - 标准 Transformer blocks
   - 自注意力机制
   - 全局信息捕捉

3. 可扩展设计
   - 可以简单缩放
   - 深度、宽度、分辨率
```

## 数学原理

### 4.1 Patch Embedding

```
将图像转换为序列:

输入：x (H×W×C)
Patch 大小：p×p

步骤:
1. 切分：(H/p) × (W/p) 个 patches
2. 展平：每个 patch → p²×C 向量
3. 投影：Linear(p²×C → D)

输出：N 个 D 维向量
其中 N = (H/p) × (W/p)

例如:
- 输入：256×256×3
- Patch: 2×2
- 输出：16384 个 D 维向量
```

### 4.2 Transformer Block

```
标准 Transformer Block:

输入：h

1. LayerNorm
   h' = LN(h)

2. Multi-Head Self-Attention
   h'' = MHSA(h') + h

3. LayerNorm
   h''' = LN(h'')

4. MLP
   h_out = MLP(h''') + h''

其中:
MHSA(Q, K, V) = softmax(QK^T/√d)V
MLP(x) = GELU(W₁x)W₂
```

### 4.3 时间嵌入

```
时间步编码:

1. Sinusoidal 位置编码
   e_t = [sin(ω₁t), cos(ω₁t), ..., sin(ω_d t), cos(ω_d t)]
   ω_i = 10000^(-2i/d)

2. MLP 投影
   emb = MLP(e_t)

3. 添加到 Transformer
   - 方式 1: 加到每个 token
   - 方式 2: 加到 layer norm 之前
   - 方式 3: adaLN (DiT 使用)
```

### 4.4 adaLN (Adaptive LayerNorm)

```
DiT 的创新设计:

传统 LayerNorm:
y = γ · (x - μ) / σ + β

adaLN:
γ, β 由时间嵌入预测
γ = W_γ(emb)
β = W_β(emb)

y = γ(emb) · (x - μ) / σ + β(emb)

好处:
- 条件信息直接注入
- 比简单相加更有效
- 类似 FiLM 的思想
```

## 训练方法

### 6.1 训练算法

```
Algorithm 1: DiT 训练

Input: 图像数据集 {x⁽¹⁾, ..., x⁽ᴺ⁾}, T, 学习率 η
Initialize: DiT 参数 θ

repeat
    # 采样一个 batch
    {x⁽¹⁾, ..., x⁽ᴹ⁾} ~ 数据集
    
    for each x⁽ⁱ⁾ in batch:
        # 随机采样时间步
        t ~ Uniform(1, T)
        
        # 采样噪声
        ε ~ N(0, I)
        
        # 加噪
        x_t = √ᾱ_t x⁽ⁱ⁾ + √(1-ᾱ_t) ε
        
        # Patchify
        patches = PatchEmbed(x_t)
        
        # DiT 前向
        ε_θ = DiT(patches, t)
        
        # 计算损失
        L = ||ε - ε_θ||²
    
    # 反向传播
    ∇_θ L = Backprop(L)
    
    # 更新参数
    θ ← θ - η ∇_θ L
    
until 收敛
```

### 6.2 训练技巧

```
1. 混合精度训练
   - 使用 FP16
   - 节省显存
   - 加速训练

2. 梯度检查点
   - 节省显存
   - 可以训练更大模型
   - 轻微速度损失

3. EMA (指数移动平均)
   - 维护参数的指数平均
   - 改善生成质量
   - 推理时使用 EMA 参数

4. 学习率调度
   - Warmup + Cosine Decay
   - 稳定训练
   - 更好收敛
```

## 代码实现

### 8.1 Patch Embedding

```python
class PatchEmbed(nn.Module):
    def __init__(self, img_size=256, patch_size=2, in_channels=3, embed_dim=768):
        super().__init__()
        self.img_size = img_size
        self.patch_size = patch_size
        self.num_patches = (img_size // patch_size) ** 2
        
        self.proj = nn.Conv2d(
            in_channels, embed_dim,
            kernel_size=patch_size, stride=patch_size
        )
    
    def forward(self, x):
        # x: (B, C, H, W)
        x = self.proj(x)  # (B, D, H/p, W/p)
        x = x.flatten(2)  # (B, D, N)
        x = x.transpose(1, 2)  # (B, N, D)
        return x
```

### 8.2 Transformer Block

```python
class TransformerBlock(nn.Module):
    def __init__(self, dim, num_heads, mlp_ratio=4.0):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        
        self.norm2 = nn.LayerNorm(dim)
        self.mlp = nn.Sequential(
            nn.Linear(dim, int(dim * mlp_ratio)),
            nn.GELU(),
            nn.Linear(int(dim * mlp_ratio), dim),
        )
    
    def forward(self, x):
        # Self-Attention
        h = self.norm1(x)
        h, _ = self.attn(h, h, h)
        x = x + h
        
        # MLP
        h = self.norm2(x)
        h = self.mlp(h)
        x = x + h
        
        return x
```

### 8.3 DiT 模型

```python
class DiT(nn.Module):
    def __init__(self, img_size=256, patch_size=2, in_channels=3,
                 dim=768, depth=12, num_heads=12):
        super().__init__()
        
        # Patch Embedding
        self.patch_embed = PatchEmbed(img_size, patch_size, in_channels, dim)
        
        # Time Embedding
        self.time_embed = nn.Sequential(
            nn.Linear(1, dim * 4),
            nn.SiLU(),
            nn.Linear(dim * 4, dim * 6),  # 用于 adaLN
        )
        
        # Transformer Blocks
        self.blocks = nn.ModuleList([
            TransformerBlock(dim, num_heads) for _ in range(depth)
        ])
        
        # Output Projection
        self.norm_final = nn.LayerNorm(dim)
        self.out_proj = nn.Linear(dim, patch_size * patch_size * in_channels)
        
        self.img_size = img_size
        self.patch_size = patch_size
    
    def forward(self, x, t):
        # Patchify
        x = self.patch_embed(x)  # (B, N, D)
        
        # Time Embedding
        t_emb = self.time_embed(t.float().unsqueeze(-1))  # (B, D*6)
        
        # Transformer Blocks
        for block in self.blocks:
            x = block(x)
        
        # Output
        x = self.norm_final(x)
        x = self.out_proj(x)  # (B, N, p²*C)
        
        # Unpatchify
        x = self.unpatchify(x)  # (B, C, H, W)
        
        return x
    
    def unpatchify(self, x):
        # x: (B, N, p²*C)
        B, N, _ = x.shape
        p = self.patch_size
        H = W = int(N ** 0.5)
        
        x = x.reshape(B, H, W, p, p, 3)
        x = x.permute(0, 5, 1, 3, 2, 4)
        x = x.reshape(B, 3, H * p, W * p)
        
        return x
```

### 8.4 训练代码

```python
def train_dit(model, dataloader, optimizer, device, epochs=10):
    model.train()
    
    for epoch in range(epochs):
        total_loss = 0
        
        for batch_idx, (images, _) in enumerate(dataloader):
            images = images.to(device)
            optimizer.zero_grad()
            
            # 随机时间步
            t = torch.randint(0, 1000, (images.shape[0],)).to(device)
            
            # 噪声
            noise = torch.randn_like(images)
            
            # 加噪
            noisy_images = add_noise(images, noise, t)
            
            # 预测
            noise_pred = model(noisy_images, t)
            
            # 损失
            loss = F.mse_loss(noise_pred, noise)
            
            # 反向传播
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
        
        avg_loss = total_loss / len(dataloader)
        print(f'Epoch {epoch+1}: Loss = {avg_loss:.4f}')
```

## 深入理解

### 9.1 为什么 Transformer 有效？

```
直觉解释 1: 全局信息

UNet (CNN):
- 卷积只有局部感受野
- 需要多层才能捕捉全局
- 长程依赖困难

Transformer:
- Self-Attention 全局感受野
- 一层就能捕捉全局
- 长程依赖天然支持

结果:
- 全局结构更好
- 长程一致性更好
```

```
直觉解释 2: 可扩展性

CNN:
- 归纳偏置强
- 小数据效果好
- 大数据可能饱和

Transformer:
- 归纳偏置弱
- 需要大数据
- 但扩展性极好

结果:
- 大规模数据下 Transformer 更好
- 可以训练更大模型
```

### 9.2 adaLN 的设计

```
为什么 adaLN 比简单相加好？

简单相加:
h = h + emb

问题:
- emb 直接加到特征上
- 可能破坏原有特征
- 条件信息不够灵活

adaLN:
h = γ(emb) · LN(h) + β(emb)

优势:
- 通过 γ, β 调制特征
- 更灵活的条件注入
- 类似 FiLM 的思想
- 不破坏原有特征分布
```

### 9.3 Patch Size 的选择

```
Patch Size 的影响:

小 Patch (p=2):
- 序列长：16384
- 计算量大
- 空间细节多
- 效果好

大 Patch (p=8):
- 序列短：1024
- 计算量小
- 空间细节少
- 效果差

权衡:
- p=2: 追求效果
- p=4: 平衡
- p=8: 追求效率
```

### 9.4 与 ViT 的对比

```
DiT vs ViT:

相似点:
- 都使用 Patch Embedding
- 都使用 Transformer
- 都处理图像

不同点:
- ViT: 图像分类
- DiT: 图像生成

- ViT: 输出类别
- DiT: 输出噪声

- ViT: 判别式
- DiT: 生成式

关键:
- DiT 需要预测每个 pixel 的噪声
- 输出维度更高
- 需要更好的空间建模
```
