## 研究背景

### 1.1 生成模型的发展

在 VAE 之前，主要的生成模型有：

| 模型 | 优点 | 缺点 |
|------|------|------|
| **高斯混合模型 (GMM)** | 理论简单 | 表达能力有限 |
| **隐马尔可夫模型 (HMM)** | 适合序列数据 | 假设太强 |
| **受限玻尔兹曼机 (RBM)** | 表达能力强 | 训练困难 |
| **自编码器 (AE)** | 简单高效 | 不能生成新样本 |

### 1.2 核心挑战

如何学习数据的**连续隐变量表示** (continuous latent representation)，并能够：
1. **编码**: 从数据到隐变量
2. **解码**: 从隐变量到数据
3. **生成**: 从隐变量分布采样生成新数据

### 1.3 为什么需要隐变量？

```
数据 x (如图像) → 隐变量 z (低维表示) → 重建数据 x'

好处:
1. 降维：高维数据 → 低维表示
2. 去噪：去除数据中的噪声
3. 生成：从隐空间采样生成新数据
4. 插值：在隐空间插值实现平滑过渡
```

## 核心问题

### 2.1 问题定义

给定数据集 {x⁽¹⁾, x⁽²⁾, ..., x⁽ᴺ⁾}，学习一个概率模型：

```
p(x) = ∫ p(x|z) p(z) dz

其中:
- x: 观测数据 (如图像)
- z: 隐变量 (低维表示)
- p(z): 先验分布 (通常为标准正态分布)
- p(x|z): 似然函数 (解码器)
```

### 2.2 核心难点

**问题**: 后验分布 p(z|x) 难以计算

```
根据贝叶斯定理:
p(z|x) = p(x|z) p(z) / p(x)
       = p(x|z) p(z) / ∫ p(x|z') p(z') dz'

分母 p(x) 需要积分，对于复杂模型无法解析计算！
```

### 2.3 传统方法的局限

| 方法 | 思路 | 问题 |
|------|------|------|
| **MCMC 采样** | 用采样近似积分 | 收敛慢，高维困难 |
| **变分推断** | 用简单分布逼近后验 | 需要手动设计 |
| **EM 算法** | 迭代优化 | 只适合简单模型 |

## 核心贡献

### 3.1 重参数化技巧 (Reparameterization Trick)

**这是 VAE 最重要的贡献！**

#### 传统采样（不可导）
```
从 q(z|x) = N(μ, σ²) 采样:
z ~ N(μ, σ²)  ← 采样操作不可导，无法反向传播！
```

#### 重参数化（可导）
```
把随机性移到外部:
ε ~ N(0, 1)  ← 外部噪声，不需要梯度
z = μ + σ ⊙ ε  ← 确定性操作，可导！

其中:
- μ, σ: 由神经网络输出
- ε: 标准正态分布采样
- ⊙: 元素乘法
```

#### 为什么重要？
```
现在可以计算梯度:
∂L/∂μ = ∂L/∂z · ∂z/∂μ = ∂L/∂z · 1
∂L/∂σ = ∂L/∂z · ∂z/∂σ = ∂L/∂z · ε

梯度可以通过 μ, σ 反向传播到编码器！
```

### 3.2 ELBO 损失函数

**Evidence Lower Bound (证据下界)**

```
原始目标：最大化对数似然 log p(x)

变分下界:
log p(x) ≥ E_{q(z|x)}[log p(x|z)] - KL(q(z|x) || p(z))
          └─────────────┘       └────────────────┘
             重构损失              正则化项

我们最大化这个下界 (ELBO):
L(θ,φ; x) = E_{z~q_φ(z|x)}[log p_θ(x|z)] - KL(q_φ(z|x) || p(z))
```

#### 两项的含义

| 项 | 作用 | 直观理解 |
|----|------|---------|
| **重构损失** | 让解码器能重建原图 | "解码器要能还原输入" |
| **KL 散度** | 让隐变量分布接近先验 | "隐变量不要太偏离标准正态" |

### 3.3 端到端训练

```
编码器 q_φ(z|x): 输入 x → 输出 μ, σ
解码器 p_θ(x|z): 输入 z → 输出 x'

联合训练:
1. 编码器学习好的隐变量表示
2. 解码器学习从隐变量重建数据
3. 两者相互促进
```

## 数学原理

### 4.1 变分推断基础

#### 问题
计算后验分布 p(z|x) 困难，因为：
```
p(z|x) = p(x|z)p(z) / p(x)
p(x) = ∫ p(x|z)p(z) dz  ← 难以计算
```

#### 解决方案
用简单分布 q(z|x) 逼近真实后验 p(z|x)

```
最小化 KL 散度:
KL(q(z|x) || p(z|x)) = E_{z~q}[log q(z|x) - log p(z|x)]

展开后:
= E_{z~q}[log q(z|x) - log p(x|z) - log p(z) + log p(x)]
= E_{z~q}[log q(z|x) - log p(z)] - E_{z~q}[log p(x|z)] + log p(x)
= KL(q(z|x) || p(z)) - E_{z~q}[log p(x|z)] + log p(x)

整理得:
log p(x) = KL(q(z|x) || p(z|x)) + E_{z~q}[log p(x|z)] - KL(q(z|x) || p(z))

由于 KL(q(z|x) || p(z|x)) ≥ 0，所以:
log p(x) ≥ E_{z~q}[log p(x|z)] - KL(q(z|x) || p(z))

右边就是 ELBO！
```

### 4.2 KL 散度解析解

当 q(z|x) = N(μ, σ²I) 且 p(z) = N(0, I) 时：

```
KL(q(z|x) || p(z)) = -½ Σ(1 + log σᵢ² - μᵢ² - σᵢ²)

推导:
KL(N(μ,σ²) || N(0,I)) = ½[tr(Σ₂⁻¹Σ₁) + (μ₂-μ₁)ᵀΣ₂⁻¹(μ₂-μ₁) - k + ln(det(Σ₂)/det(Σ₁))]

代入 μ₁=μ, Σ₁=σ²I, μ₂=0, Σ₂=I:
= ½[tr(I·σ²I) + (0-μ)ᵀI(0-μ) - k + ln(1/det(σ²I))]
= ½[Σσᵢ² + Σμᵢ² - k - Σlog σᵢ²]
= -½ Σ(1 + log σᵢ² - μᵢ² - σᵢ²)
```

### 4.3 梯度估计

#### 原始问题
```
∇_φ E_{z~q_φ(z|x)}[f(z)] = ?

直接求导:
∇_φ ∫ q_φ(z|x) f(z) dz = ∫ ∇_φ q_φ(z|x) f(z) dz

问题：∇_φ q_φ(z|x) 难以计算！
```

#### 重参数化解决
```
z = g_φ(ε, x), 其中 ε ~ p(ε)

E_{z~q_φ(z|x)}[f(z)] = E_{ε~p(ε)}[f(g_φ(ε, x))]

∇_φ E_{ε~p(ε)}[f(g_φ(ε, x))] = E_{ε~p(ε)}[∇_φ f(g_φ(ε, x))]

现在可以用蒙特卡洛估计:
≈ 1/M Σ ∇_φ f(g_φ(ε⁽ᵐ⁾, x)), 其中 ε⁽ᵐ⁾ ~ p(ε)
```

## 模型架构

### 5.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                      VAE                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  输入 x (3×256×256)                                 │
│       ↓                                             │
│  ┌─────────────┐                                    │
│  │  编码器     │  q_φ(z|x)                          │
│  │  (Encoder)  │                                    │
│  └─────────────┘                                    │
│       ↓                                             │
│  μ, logσ² (各 128 维)                                │
│       ↓                                             │
│  重参数化: z = μ + σ⊙ε, ε~N(0,1)                    │
│       ↓                                             │
│  ┌─────────────┐                                    │
│  │  解码器     │  p_θ(x|z)                          │
│  │  (Decoder)  │                                    │
│  └─────────────┘                                    │
│       ↓                                             │
│  输出 x' (3×256×256)                                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 5.2 编码器设计

```python
编码器 q_φ(z|x):
输入：x (图像)
输出：μ, logσ² (隐变量分布参数)

典型架构:
x → Conv2d(3→32) → ReLU → Conv2d(32→64) → ReLU
  → Flatten → Linear(64×H×W→128)
  → [μ分支：Linear(128→128)]
  → [σ分支：Linear(128→128)]
```

### 5.3 解码器设计

```python
解码器 p_θ(x|z):
输入：z (隐变量)
输出：x' (重建图像)

典型架构:
z → Linear(128→64×H×W) → Unflatten
  → ConvTranspose2d(64→32) → ReLU
  → ConvTranspose2d(32→3) → Sigmoid
  → x'
```

### 5.4 隐空间维度选择

| 数据集 | 图像大小 | 推荐隐变量维度 |
|--------|---------|---------------|
| MNIST | 28×28 | 2-32 |
| CIFAR-10 | 32×32 | 32-128 |
| ImageNet | 256×256 | 128-512 |

**原则**: 隐变量维度应远小于输入维度，但要足够表达关键信息

## 训练方法

### 6.1 损失函数

```python
def vae_loss(x, x_recon, mu, logvar):
    # 重构损失 (MSE 或 BCE)
    recon_loss = F.binary_cross_entropy(x_recon, x, reduction='sum')
    
    # KL 散度
    kl_loss = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    
    # 总损失
    return recon_loss + kl_loss
```

### 6.2 训练流程

```
Algorithm 1: VAE 训练

Input: 数据集 {x⁽¹⁾, ..., x⁽ᴺ⁾}, 学习率 η, batch size M
Initialize: 编码器参数 φ, 解码器参数 θ

repeat
    # 采样一个 batch
    {x⁽¹⁾, ..., x⁽ᴹ⁾} ~ 数据集
    
    for each x⁽ⁱ⁾ in batch:
        # 前向传播
        μ⁽ⁱ⁾, σ⁽ⁱ⁾ = Encoder(x⁽ⁱ⁾)
        ε⁽ⁱ⁾ ~ N(0, I)
        z⁽ⁱ⁾ = μ⁽ⁱ⁾ + σ⁽ⁱ⁾ ⊙ ε⁽ⁱ⁾  # 重参数化
        x'⁽ⁱ⁾ = Decoder(z⁽ⁱ⁾)
        
        # 计算损失
        L⁽ⁱ⁾ = -log p_θ(x⁽ⁱ⁾|z⁽ⁱ⁾) + KL(q_φ(z|x⁽ⁱ⁾) || p(z))
    
    # 平均损失
    L = 1/M Σ L⁽ⁱ⁾
    
    # 反向传播
    ∇_θ L, ∇_φ L = Backprop(L)
    
    # 更新参数
    θ ← θ - η ∇_θ L
    φ ← φ - η ∇_φ L
    
until 收敛
```

### 6.3 训练技巧

| 技巧 | 说明 | 效果 |
|------|------|------|
| **KL Annealing** | 逐步增加 KL 权重 | 避免 posterior collapse |
| **Free Bits** | KL 损失设置阈值 | 改善生成质量 |
| **BatchNorm** | 使用批归一化 | 加速收敛 |
| **Learning Rate Warmup** | 学习率预热 | 稳定训练 |

## 代码实现

### 8.1 PyTorch 实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, latent_dim=128):
        super().__init__()
        
        # 编码器
        self.encoder = nn.Sequential(
            nn.Conv2d(3, 32, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.Conv2d(32, 64, 3, stride=2, padding=1),
            nn.ReLU(),
            nn.Flatten(),
        )
        
        self.fc_mu = nn.Linear(64*64*64, latent_dim)
        self.fc_logvar = nn.Linear(64*64*64, latent_dim)
        
        # 解码器
        self.decoder_fc = nn.Linear(latent_dim, 64*64*64)
        self.decoder = nn.Sequential(
            nn.Unflatten(1, (64, 64, 64)),
            nn.ConvTranspose2d(64, 32, 3, stride=2, padding=1, output_padding=1),
            nn.ReLU(),
            nn.ConvTranspose2d(32, 3, 3, stride=2, padding=1, output_padding=1),
            nn.Sigmoid(),
        )
    
    def encode(self, x):
        h = self.encoder(x)
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)
        return mu, logvar
    
    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)
        return mu + eps * std
    
    def decode(self, z):
        h = self.decoder_fc(z)
        return self.decoder(h)
    
    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterize(mu, logvar)
        x_recon = self.decode(z)
        return x_recon, mu, logvar
    
    def loss(self, x, x_recon, mu, logvar):
        # 重构损失
        recon_loss = F.binary_cross_entropy(x_recon, x, reduction='sum')
        # KL 损失
        kl_loss = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
        return recon_loss + kl_loss
```

### 8.2 训练代码

```python
def train_vae(model, dataloader, optimizer, device, epochs=10):
    model.train()
    
    for epoch in range(epochs):
        total_loss = 0
        
        for batch_idx, (x, _) in enumerate(dataloader):
            x = x.to(device)
            optimizer.zero_grad()
            
            # 前向传播
            x_recon, mu, logvar = model(x)
            
            # 计算损失
            loss = model.loss(x, x_recon, mu, logvar)
            
            # 反向传播
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
        
        avg_loss = total_loss / len(dataloader.dataset)
        print(f'Epoch {epoch+1}: Loss = {avg_loss:.4f}')
```

### 8.3 生成样本

```python
@torch.no_grad()
def generate_samples(model, n_samples=16, device='cuda'):
    model.eval()
    
    # 从先验分布采样
    z = torch.randn(n_samples, model.fc_mu.out_features).to(device)
    
    # 解码生成
    samples = model.decode(z)
    
    return samples
```

## 深入理解

### 9.1 为什么预测分布而不是点估计？

**传统 AE**: 编码器输出一个点 z
**VAE**: 编码器输出一个分布 N(μ, σ²)

**好处**:
1. **正则化**: KL 散度防止过拟合
2. **生成能力**: 可以从分布采样生成新数据
3. **不确定性**: σ 表示模型的不确定性

### 9.2 重参数化的本质

```
重参数化不是技巧，而是思想转变：

传统思路：学习从 x 到 z 的确定性映射
VAE 思路：学习从 x 到分布参数的映射，然后采样

关键洞察：
- 采样操作本身不可导
- 但采样的参数 (μ, σ) 可以参与梯度计算
- 把随机性移到外部，内部保持确定性
```

### 9.3 ELBO 的直观理解

```
ELBO = 重构损失 - KL 散度

可以理解为：
- 重构损失：解码器要能还原输入（保真度）
- KL 散度：隐变量要接近先验分布（正则化）

两者权衡：
- 只优化重构：退化为普通 AE，不能生成
- 只优化 KL：隐变量变成标准正态，丢失信息
- 平衡两者：既能重建，又能生成
```

### 9.4 隐空间的几何结构

```
VAE 学到的隐空间特点：

1. 连续性：相近的 z 产生相近的 x
2. 完整性：从隐空间采样都能产生合理数据
3. 可解释性：不同维度可能对应不同语义

示例：
- MNIST: 某些维度控制数字粗细，某些控制倾斜
- 人脸：某些维度控制性别，某些控制表情
```

## 局限性与改进

### 10.1 主要局限

| 问题 | 表现 | 原因 |
|------|------|------|
| **模糊问题** | 生成图像模糊 | MSE/BCE 损失导致平均化 |
| **Posterior Collapse** | KL 散度接近 0 | 解码器太强，忽略隐变量 |
| **隐空间结构** | 隐空间可能不连续 | 高斯假设太强 |

### 10.2 改进方法

| 改进 | 方法 | 效果 |
|------|------|------|
| **VQ-VAE** | 向量量化隐变量 | 解决模糊问题 |
| **β-VAE** | 增加 KL 权重 | 更好的解耦 |
| **Cyclical VAE** | 周期性 KL 退火 | 避免 posterior collapse |
| **Flow-based VAE** | 用 Normalizing Flow | 更灵活的后验 |

## 与 Diffusion 的关系

### 11.1 理论联系

```
VAE: 编码一次 → 解码一次
Diffusion: 编码 T 次 (加噪) → 解码 T 次 (去噪)

可以理解为：
- Diffusion 是"多层次"的 VAE
- 每个时间步 t 相当于一个 VAE
- T 个 VAE 串联形成 Diffusion
```

### 11.2 架构对比

| 特性 | VAE | Diffusion |
|------|-----|-----------|
| **隐变量** | 单次采样 z | 多次采样 x₁, x₂, ..., x_T |
| **编码** | 一次前向 | T 步加噪 |
| **解码** | 一次前向 | T 步去噪 |
| **训练** | ELBO | 去噪分数匹配 |
| **生成质量** | 较模糊 | 清晰 |
| **采样速度** | 快 | 慢 (但可加速) |

### 11.3 为什么 Diffusion 效果更好？

1. **多层次**: 从粗到细逐步生成
2. **简单目标**: 只预测噪声，比 VAE 预测整个图像简单
3. **训练稳定**: 不像 GAN 那样容易崩溃
