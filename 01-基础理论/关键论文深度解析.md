## 🔴 必读经典 (按阅读顺序)

### 1. VAE (2013) - 变分自编码器

**论文**: Kingma & Welling. "Auto-Encoding Variational Bayes" (ICLR 2014)  
**链接**: https://arxiv.org/abs/1312.6114

#### 核心问题
- 如何学习数据的潜在表示 (latent representation)?
- 如何在生成模型中进行有效的贝叶斯推断？

#### 核心创新
1. **重参数化技巧 (Reparameterization Trick)**
   ```
   传统采样：z ~ N(μ, σ²)  ← 不可导
   重参数化：ε ~ N(0, 1), z = μ + σ·ε  ← 可导!
   ```

2. **ELBO 损失函数**
   ```
   L(θ,φ; x) = E_q[log p(x|z)] - KL(q(z|x) || p(z))
              重构损失           正则化项
   ```

#### 为什么重要
- 为 Diffusion 模型奠定了理论基础
- Diffusion 可以看作"多个 VAE 串联"
- 隐空间 (latent space) 概念的起源

#### 个人理解
> VAE 的核心思想是"编码 - 解码"，但加入了概率分布的约束。编码器不直接输出 latent code，而是输出分布的参数 (μ, σ)，然后从分布中采样。这样学到的 latent space 是连续的、平滑的，适合生成任务。

### 2. DDPM (2020) - 去噪扩散概率模型

**论文**: Ho et al. "Denoising Diffusion Probabilistic Models" (NeurIPS 2020)  
**链接**: https://arxiv.org/abs/2006.11239

#### 核心问题
- 如何生成高质量图像？
- GAN 训练不稳定，有没有替代方案？

#### 核心创新

**1. 前向过程 (加噪)**
```
q(x_t | x_{t-1}) = N(x_t; √(1-β_t)·x_{t-1}, β_t·I)
q(x_t | x_0) = N(x_t; √ᾱ_t·x_0, (1-ᾱ_t)·I)  ← 任意时刻 t 可直接计算!
```

**2. 反向过程 (去噪)**
```
p_θ(x_{t-1} | x_t) = N(x_{t-1}; μ_θ(x_t, t), Σ_θ(x_t, t))
```

**3. 关键洞察：预测噪声而非原图**
```
损失函数：L = E[||ε - ε_θ(x_t, t)||²]
预测目标：噪声 ε，而不是原图 x_0
```

#### 为什么有效
1. **分解复杂任务**: 将"生成图像"分解为 1000 个简单的"去噪一步"
2. **训练稳定**: 不像 GAN 那样需要博弈平衡
3. **覆盖全面**: 生成样本多样性好，不易 mode collapse

#### 技术细节
- **噪声调度 (Noise Schedule)**: β_t 从 10⁻⁴ 线性增加到 0.02
- **网络架构**: 简单的 UNet，时间 t 作为嵌入输入
- **采样步数**: 原始 1000 步，后续可加速到 50-100 步

#### 个人理解
> DDPM 的反直觉之处在于"预测噪声"。我的理解是：噪声比原图更简单 (高斯分布)，学习去除噪声比学习直接生成原图更容易。这就像"从混乱中创造秩序"比"凭空创造秩序"更容易。

### 3. DDIM (2020) - 加速推理

**论文**: Song et al. "Denoising Diffusion Implicit Models" (ICLR 2021)  
**链接**: https://arxiv.org/abs/2010.02502

#### 核心问题
- DDPM 需要 1000 步采样，太慢了！
- 能否减少采样步数？

#### 核心创新

**1. 非马尔可夫链**
```
DDPM: x_t → x_{t-1} → ... → x_0  (马尔可夫)
DDIM: x_t → x_{t-Δ} → ... → x_0  (跳步！)
```

**2. 确定性采样**
```
DDPM 是随机过程，DDIM 是确定性的
相同起点 → 相同结果 (可复现)
```

**3. 统一框架**
```
DDIM 包含 DDPM 作为特例
通过调节 η 参数，可以在两者之间插值
```

#### 加速效果
| 采样步数 | FID (越低越好) | 速度提升 |
|---------|---------------|---------|
| 1000 (DDPM) | 3.17 | 1x |
| 100 (DDIM) | 4.61 | 10x |
| 50 (DDIM) | 5.42 | 20x |
| 20 (DDIM) | 8.33 | 50x |

#### 个人理解
> DDIM 的核心洞察是：扩散过程的训练目标不依赖于马尔可夫假设。只要边际分布 q(x_t|x_0) 不变，中间过程可以任意设计。这就像"条条大路通罗马"，DDPM 走的是曲折小路，DDIM 走的是直线高速。

### 4. LDM / Stable Diffusion (2021) - 隐空间扩散

**论文**: Rombach et al. "High-Resolution Image Synthesis with Latent Diffusion Models" (CVPR 2022)  
**链接**: https://arxiv.org/abs/2112.10752

#### 核心问题
- DDPM 在像素空间训练，计算量太大
- 如何生成高分辨率图像 (如 1024x1024)？

#### 核心创新

**1. 两阶段训练**
```
阶段 1: 训练 VAE，学习压缩图像的 latent representation
阶段 2: 在 latent space 训练 Diffusion 模型
```

**2. 感知压缩 (Perceptual Compression)**
```
像素空间：512×512×3 = 786,432 维
隐空间：64×64×4 = 16,384 维  ← 压缩 48 倍!
```

**3. Cross-Attention 条件机制**
```
文本 → CLIP Text Encoder → 文本嵌入
图像隐变量 + 文本嵌入 → Cross-Attention → 条件生成
```

#### 为什么是里程碑
1. **计算效率**: 训练成本降低 48 倍，消费级显卡可运行
2. **开源生态**: 代码和权重完全开源，引爆社区创新
3. **商业化**: 催生 Stability AI，AI 绘画进入大众视野

#### 架构细节
```
VAE Encoder:  图像 (512×512×3) → 隐变量 (64×64×4)
Diffusion UNet: 处理隐变量，8 个 ResNet 块 + Cross-Attention
VAE Decoder:  隐变量 (64×64×4) → 图像 (512×512×3)
```

#### 个人理解
> LDM 的智慧在于"降维打击"。像素空间有很多冗余信息 (相邻像素高度相关)，在隐空间去噪就像"在压缩包里修改"，效率自然高。Cross-Attention 的设计也很优雅，让文本和图像在 latent space 对话。

---

### 5. DiT (2022) - Transformer 取代 UNet

**论文**: Peebles & Xie. "Scalable Diffusion Models with Transformers" (ICCV 2023)  
**链接**: https://arxiv.org/abs/2212.09748

#### 核心问题
- UNet 是 CNN 架构，扩展性有限
- Transformer 在 NLP 成功，能否用于 Diffusion？

#### 核心创新

**1. 纯 Transformer 架构**
```
输入：图像 patches (类似 ViT)
架构：标准 Transformer blocks
输出：去噪后的 patches
```

**2. 可扩展性**
```
DiT-XL/2:  675M 参数
DiT-XL/4:  307M 参数
DiT-XL/8:  119M 参数
```

**3. 关键发现**
- Transformer 块数越多，性能越好
- 注意力头数对性能影响不大
- 可以简单缩放，性能可预测

#### 为什么重要
1. **架构统一**: 图像生成和 NLP 使用相同架构
2. **性能超越**: DiT-XL/2 超越最佳 UNet 模型
3. **后续影响**: SD3、FLUX 都采用 DiT 架构

#### 个人理解
> DiT 的成功证明了 Transformer 的通用性。UNet 的归纳偏置 (局部性、平移不变性) 对 Diffusion 不是必需的。Transformer 的全局注意力机制反而更适合捕捉图像的长程依赖。

## 🟡 进阶必读

### 6. ControlNet (2023) - 条件控制

**论文**: Zhang et al. "Adding Conditional Control to Text-to-Image Diffusion Models" (ICCV 2023)  
**链接**: https://arxiv.org/abs/2302.05543

#### 核心问题
- SD 只能文本控制，无法精确控制构图
- 如何用边缘图、深度图、姿态图等控制生成？

#### 核心创新

**1. 锁定 + 复制策略**
```
锁定原始 SD 模型参数 (不更新)
复制一份 encoder 层，训练条件控制
```

**2. Zero Convolution**
```
初始化为零卷积：输出全为 0
逐步学习：逐渐注入条件信息
避免破坏原始模型能力
```

**3. 通用框架**
```
支持多种条件：Canny 边缘、深度图、姿态、涂鸦等
一个框架，多种 ControlNet 变体
```

#### 应用价值
- **工业级应用**: 建筑设计、线稿上色、姿势控制
- **生态爆发**: Civitai 上数千个 ControlNet 模型
- **开源贡献**: 完全开源，社区快速跟进

#### 个人理解
> ControlNet 的优雅在于"不破坏原模型"。通过 zero convolution 初始化为 0，确保训练初期模型行为不变，然后逐步学习条件控制。这就像"给汽车加装导航"，不改变发动机，但增加了方向控制。

### 7. LoRA (2021) - 高效微调

**论文**: Hu et al. "LoRA: Low-Rank Adaptation of Large Language Models" (ICLR 2022)  
**链接**: https://arxiv.org/abs/2106.09685

#### 核心问题
- 大模型全参数微调成本太高
- 能否只微调少量参数？

#### 核心创新

**1. 低秩分解**
```
原始权重更新：ΔW (d×k，参数量大)
LoRA 分解：ΔW = B·A  (B: d×r, A: r×k, r << d)
r=8 时，参数量减少 1000 倍!
```

**2. 冻结 + 旁路**
```
冻结原始模型权重 W
训练 LoRA 权重 A, B
推理时：W' = W + B·A
```

**3. 无推理延迟**
```
训练完成后可合并：W' = W + B·A
推理时无额外计算开销
```

#### 在 SD 中的应用
```
2023.01: Hugging Face 将 LoRA 引入 Stable Diffusion
效果：用几千张图、几小时训练，实现风格微调
生态：Civitai 上数十万 LoRA 模型
```

#### 个人理解
> LoRA 的洞察是"权重更新是低秩的"。大模型微调时，真正变化的方向很少，大部分权重几乎不变。LoRA 就像"给模型戴眼镜"，不改变大脑，只调整"看世界的方式"。

## 🟢 前沿探索

### 8. Rectified Flow / Flow Matching (2022) - 流模型

**论文**: 
- Liu et al. "Flow Straight and Fast" (ICLR 2023): https://arxiv.org/abs/2209.03003
- Lipman et al. "Flow Matching for Generative Modeling" (ICLR 2023): https://arxiv.org/abs/2210.02747

#### 核心问题
- Diffusion 的噪声路径是弯曲的，采样效率低
- 能否让生成路径更直、更快？

#### 核心创新

**1. 直线轨迹**
```
Diffusion: x_0 → 弯曲路径 → x_1
Flow:      x_0 → 直线路径 → x_1
```

**2. 常微分方程 (ODE)**
```
dx/dt = v(x, t)  ← 学习速度场
从噪声积分到数据：x_1 = x_0 + ∫v(x,t)dt
```

**3. 更少采样步数**
```
Diffusion: 100-1000 步
Flow: 10-50 步即可
```

#### 为什么是未来
1. **效率**: 采样速度快 10-20 倍
2. **质量**: 生成质量相当或更好
3. **采用**: SD3、FLUX 都采用 Flow Matching

#### 个人理解
> Flow Matching 的直觉是"走直线最近"。Diffusion 像醉汉走路 (随机游走)，Flow 像导航开车 (直线前进)。数学上，Flow 学习的是最优传输路径，自然更高效。

### 9. MMDiT (2024) - SD3 架构

**论文**: Esser et al. "Scaling Rectified Flow Transformers for High-Resolution Image Synthesis"  
**链接**: https://arxiv.org/abs/2403.03206

#### 核心创新

**1. 多模态 DiT**
```
传统 DiT: 图像 patches → Transformer → 输出
MMDiT:   图像 patches + 文本 tokens → 联合处理 → 输出
```

**2. 双塔架构**
```
图像塔：处理图像 patches
文本塔：处理文本 tokens
联合注意力：图像 - 文本交互
```

**3. 流匹配 + DiT**
```
结合 Flow Matching 的高效采样
结合 DiT 的可扩展性
```

#### 实际表现
- **文字渲染**: 能准确生成英文单词
- **语义理解**: 复杂提示词遵循度提升
- **分辨率**: 原生支持 2048×2048+

#### 争议
- SD3 Medium 发布时有人体崩坏问题
- 许可证苛刻，社区反弹
- SD3.5 修复了大部分问题

### 10. FLUX 架构解析 (2024)

**技术报告**: Black Forest Labs. "FLUX.1 Technical Report"  
**链接**: https://github.com/black-forest-labs/flux

#### 核心创新

**1. 流匹配 + DiT + RoPE**
```
Flow Matching: 高效采样
DiT: 可扩展架构
RoPE (旋转位置编码): 更好的位置表示
```

**2. 并行注意力**
```
传统：自注意力 → 交叉注意力 (串行)
FLUX:  自注意力 + 交叉注意力 (并行)
速度提升：1.5-2x
```

**3. 提示词遵循度**
```
能准确生成复杂文字
手指数量正确
空间关系理解准确
```

#### 为什么超越 SD3
1. **团队**: 原 SD 核心团队 (Robin Rombach 等)
2. **数据**: 更高质量的训练数据
3. **工程**: 更好的训练稳定性和调优
